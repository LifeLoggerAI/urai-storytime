URAI StoryTime
Read This Before You Touch Anything
StoryTime is intentionally small.

It is designed to feel calm, repetitive, and non-demanding. This is not a missing feature — it is the point.

What You Must Not Add
Do not add:

Analytics or tracking
Personalization
Scores, streaks, or progress
Child-facing controls
Emotional analysis
Feature flags
Roadmaps
If you think one of these is needed, you are probably solving the wrong problem.

What You May Do
You may:

Fix bugs that break calm or silence
Improve infrastructure reliability
Write new stories (content-only)
Nothing else.

The Lock
This repo contains a STORYTIME_LOCK.md file.

If a change violates that document, it must not be merged.

No exceptions.

Philosophy (Non-Negotiable)
StoryTime exists to witness, not shape.

It does not optimize behavior. It does not explain emotions. It does not compete for attention.

If a change makes the system louder, faster, or more impressive — it is wrong.

When in Doubt
Do less. Wait longer. Remove instead of add.

If you are looking to build something bigger, this may not be the right project.
